# Evolveme Growth — 版本控制规范

## 版本号格式

```
V{主}.{次}.{功能}.{构建}
```

| 段位 | 含义 | 升级权限 | 示例 |
|------|------|----------|------|
| 主（第1位） | 架构重构、技术栈更换 | 仅船长批准 | V1 → V2 |
| 次（第2位） | 重大功能模块新增 | 仅船长批准 | V1.0 → V1.1 |
| 功能（第3位） | 功能模块完成/重构 | 仅船长批准 | V1.0.0 → V1.0.1 |
| 构建（第4位） | Bug修复、小改动、日常迭代 | 自动递增 | V1.0.0.0 → V1.0.0.1 |

## 规则

1. **起始版本**：V1.0.0.0（2026-03-28，UI骨架 + 服务层封装）
2. **自动递增**：每次提交仓库前，构建号 +1
3. **升代需批准**：倒数第二位及之前的数字段变更，必须经船长明确批准
4. **提交流程**：开发 → 自检 → 审计（E2E/真机测试）→ 船长批准 → 版本号更新 → Git commit + push
5. **Git commit 消息格式**：`V{版本号}: {简要描述}`
